案例

自定义镜像mycentosjava17

要求

使Centos7镜像具备vim+ifconfig+jdk17

jdk下载官网：https://www.oracle.com/java/technologies/downloads/

编写

准备编写Dockerfile文件（文件名的D要大写）

  • 在数据机根目录下创建目录myfile

  • 将openjdk-17.0.1_linux-x64_bin.tar.gz文件放在该目录下

  • 在该目录下创建Dockerfile文件，并进行文件内容的编写

  • # 基础镜像，当前新镜像是基于哪个镜像的，指定一个已经存在的镜像作为模板，第一条必须是from
FROM centos
# 镜像维护者的姓名和邮箱地址
MAINTAINER summer<3311415973@qq.com>

# 登录后，进入usr/local这个目录（key-value）
ENV MYPATH /usr/local
# 指定在创建容器后，终端默认登录进来的工作目录，一个落脚点
WORKDIR $MYPATH
 
# RUN是容器构建时需要运行的命令(有两种格式：shell格式、exec格式)
# shell格式：等同于直接在终端操作shell命令
# exec格式：RUN["./test.php","dev","offline"] 等价于 RUN ./test.php dev offline
#安装vim编辑器(shell格式)
RUN yum -y install vim
#安装ifconfig命令查看网络IP
RUN yum -y install net-tools
#安装java8及lib库
RUN yum -y install glibc.i686 # 在安装jdk之前，先安装一个特定的程序库
RUN mkdir /usr/local/java
# ADD 将宿主机目录下的文件拷贝进镜像且会自动处理URL和解压tar压缩包
#ADD 是相对路径jar,把jdk-8u171-linux-x64.tar.gz添加到容器中,（这里）安装包必须要和Dockerfile文件在同一位置（解压后放在/usr/local/java/目录下）
ADD jdk-17_linux-x64_bin.tar.gz /usr/local/java/
#配置java环境变量
ENV JAVA_HOME /usr/local/java/jdk-17
# ENV JRE_HOME $JAVA_HOME/jre
ENV CLASSPATH # $JAVA_HOME/lib/dt.jar:$JAVA_HOME/lib/tools.jar:$JRE_HOME/lib:$CLASSPATH
$JAVA_HOME/lib/dt.jar:$JAVA_HOME/lib/tools.jar:$JRE_HOME/lib:$CLASSPATH
ENV PATH $JAVA_HOME/bin:$PATH
 
EXPOSE 80
 
CMD echo $MYPATH
CMD echo "success--------------ok"
CMD /bin/bash

报错：

在构建时，出现如下错误，该怎么处理？

error: failed to download metadata for repo ‘appstream’: cannot prepare internal mirrorlist: no urls in mirrorlist

解决错误：

这个错误通常是由于系统无法访问软件源或软件源配置文件出现问题导致的。以下是一些可能的解决方案：

  1. 检查网络连接是否正常，确保可以访问软件源。
  2. 尝试清除缓存并重新运行更新命令，例如sudo yum clean all && sudo yum update。
  3. 确认软件源配置文件中没有任何语法错误或拼写错误，并尝试使用另一个软件源。
  4. 再次运行更新命令，并在命令后面添加-v选项，以获取更多详细信息，例如sudo yum update -v。

如果上述方法都无法解决问题，请考虑采用其他方法，如手动下载并安装所需的软件包。
